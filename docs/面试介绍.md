# AI Test Copilot — 项目介绍

## 一句话概述

**AI 驱动的测试用例自动生成与智能评审平台** —— 上传需求文档，自动产出结构化测试用例和 Excel，生成后 AI 自评审并标记问题。

---

## 解决了什么问题

测试工程师读需求文档 → 手写测试用例 → 整理 Excel，纯人工流程耗时长、质量参差不齐。这个项目把整个流程自动化：**需求文档进去，带评审的测试用例 Excel 出来**。

---

## 技术架构

```
Streamlit 前端 ←→ FastAPI 后端 ←→ DeepSeek AI
                     ↕
                 SQLite
```

| 层 | 技术选型 | 理由 |
|----|---------|------|
| 前端 | Streamlit | Python 原生，无需写 JS，快速交付 |
| 后端 | FastAPI + 异步 | 高并发 AI 调用，async/await 原生支持 |
| AI | DeepSeek API | 兼容 OpenAI SDK，性价比高 |
| 数据校验 | Pydantic v2 | AI 输出不可控，必须严格校验 |
| 存储 | SQLite + aiosqlite | MVP 轻量，异步驱动 |

## 核心功能

### 1. 功能点提取
文档分块 → 逐块调用 AI → Pydantic 校验 → 去重 → 入库。失败自动重试（最多 3 次，带错误反馈引导 AI 自修正）。

### 2. 测试点生成
强制四维度覆盖（功能/异常/安全/边界），缺失任一类直接触发 `ValidationError` 重试。8 并发提升速度。

### 3. 测试用例生成
小功能点合并批处理减少 API 调用次数，10 并发。用例类型（正向/逆向/边界）自动推断。

### 4. 生成后自动评审
两层评审机制：
- **本地快速检查**：标题长度、步骤数量、编号格式、预期结果
- **AI 深度评审**（抽样 15 条）：步骤原子化、预期可验证性、覆盖率平衡
- 问题直接标记在用例列表上，用户可原地修改/删除

### 5. 一键生成 + Excel 导出
全流程一键触发。Excel 8 列标准格式，优先级 P0-P3 颜色标记，冻结表头。

## 技术亮点

### AI 调用可靠性
- `chat_json()` 自研 JSON 修复器：markdown 清洗 → 边界提取 → 尾随逗号修复 → 截断补全
- 网络层指数退避重试 + 语义层 Pydantic 校验重试
- 所有 AI 端点防重复提交（状态检查 + 事务提交）

### 并发优化
- 测试点生成 8 并发，测试用例生成 10 并发
- ≤3 个测试点的功能点 5 合一批量调用 LLM，减少 60% API 调用
- 线程池执行同步 AI 调用，避免阻塞事件循环

### 安全加固
- IP 限流中间件（自定义实现，跳过 health/docs 端点）
- 路径遍历防护（`resolve()` 验证路径在合法目录内）
- CORS 白名单 + 异常信息脱敏
- 文件上传大小限制 + `.env`/`.db` gitignore 排除

### 代码质量
- V1→V2 服务演进：dataclass → Pydantic BaseModel，新增校验+重试
- 前后端解耦：services 层独立于 FastAPI，可脱离 Web 框架单独调用
- Prompt 模板外置：与代码解耦，修改 Prompt 无需改逻辑
